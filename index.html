    What is Reflect?

    Reflect is a companion tool for the JINI Library used to automatically generate the binding between Java and C++. The tool can create JINI bindings for given class or classes. It accepts the classes in java.lang.Class form to generate binding for that one class only or java.lang.* to generate it for all java.lang classes (but not java.class.reflect classes). When an internal class is provided (that is, when $ is present), it will be replaced with the outer-most class.

    Synopsis

    java -jar reflect.jar [-h] -a API [-8] [--dest DIR] [--inc DIR] [--src DIR]
                      [--preserve-refs] [--parents] [--all-deps] [--all]
                      [--file-list FILE] [CLASS [CLASS ...]]

    Positional arguments:

    • CLASS : class and/or package to generate bindings for.

    Android API Arguments:

    • -a API, --android API : Android API Level (e.g. -a 17 for Android 4.2).

    Optional arguments:

    • -h, --help : show the help message and exit
    • -8, --utf8 : replace java.lang.String with const char* (default: false)
    • --dest DIR : the output directory (default: .\code)
    • --inc DIR : the output directory for .hpp files (default: $dest\inc)
    • --src DIR : the output directory for .cpp files (default: $dest\src)
    • --preserve-refs : when Reflect encounters a type it shouldn't know about, it removes the declaration the type is in from the binding; the list of types Reflect should know about consists of builtin types, members of the java.lang package and the classes given as positional arguments; this turns this behaviour off (default: false)
    • --parents : generate bindings for the super class and interfaces classes (default: false)
    • --all-deps : generate bindings for the super class, interfaces, field and parameter classes; implies --parent and --preserve-refs (default: false)
    • --all : Generate bindings for all the classes in the API; implies --all-deps, --parent and --preserve-refs (default: false)
    • --file-list FILE : output list of generated files for further processing

    Examples

    Help

    $ java -jar path/to/reflect.jar --help

    Minimal Bitmap binding

    To generate Android bindings big enough to manipulate bitmap's pixmap and blend it on the canvas:

    $ java -jar path/to/reflect.jar -a 17 \
> --parents \
> android.graphics.Bitmap \
> android.graphics.Canvas \
> android.graphics.Paint

    This will create headers and code for both Bitmap and Canvas and for their super classes up to and including Object. It will generate binding for the drawBitmap(Bitmap,float,float,Paint) to be able to blend the bitmap. This will not, however, generate binding for getScaledHeight(DisplayMetrics), as neither android.util.DisplayMetrics, --preserve-refs nor --all-deps was given as an argument.

    To generate java.lang package for the JINI project:

    $ java -jar path/to/reflect.jar -a 17 \
> -8 --all-deps java.lang.*

    API Documentation

    You can create Javadocs with javadoc:aggregate mojo after cloning the repository:

    $ mvn javadoc:aggregate

    This will generate entire javadoc in ./target/site/apidocs. Alternatively, you can go to online docs.
