com.mbits.plugins

Plugins Manager base. If you want to use it in your project, you should provide your plugins vendors with the plugin interface and the location, where the jars should be placed. For an example, let's see a plugin system allowing to perform foo() from the plugins.

FooPlugin.java

This is the interface class that must be known to the plugin vendors. 

package com.example.plugins;

import com.mbits.plugins.Plugin;
 
public interface FooPlugin extends Plugin {
    void foo();
}

FooPlugins.java

Here you prepare the entire environment for the Plugin Manager. First, you implement your own Plugins.Impl providing any pass-through methods from you application code to the plugins. Then, you may think about providing your version of loadPlugins hiding the interface class from the rest of the application. You may also provide code locating the plugins directory here. 

package com.example.plugins;
 
import java.io.File;
import com.mbits.plugins.Plugins;

public class FooPlugins extends Plugins {

    private static class Impl extends Plugins.Impl {
        public void foo() {
            for (FooPlugin plug: m_plugins) {
                plug.foo();
            }
        }
    }

    private static Impl impl = null;
    private static boolean hasImpl() {
        if (impl == null)
            impl = new Impl();
        return impl != null;
    }

    public static void loadPlugins(File file) {
        if (!hasImpl()) return;
        try {
            impl.loadPlugins(file, FooPlugin.class);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    public static void foo() {
        if (!hasImpl()) return;
        impl.foo();
    }
}

SuperFooPlugin.java

This is an implementation of a FooPlugin. The code should be packed into the jar file and placed into plugins directory. 

package com.example.superfoo;

import com.example.plugins.FooPlugin;

public class SuperFooPlugin implements FooPlugin {
    public SuperFooPlugin() {
        //...
    }

    @Override
    public String getName() { return "SuperFoo"; }

    @Override
    void foo() {
        //...
    }
}

This class must be packed into a Jar file. For the Manager to find your plugin, you must add Plugin-Class entry to the manifest file: 

Plugin-Class: com.example.superfoo.SuperFooPlugin
