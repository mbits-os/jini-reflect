Public Methods

public void loadPlugins (File dir, Class<T> clazz)

Loads the plugin into m_plugins. Enumerates the Jar files, first looking for the manifest entry called Plugin-Class, then tries to load the specified class, but only if the class is implementing the clazz interface. The work is performed by loadPlugin called through loadPluginWrapper.

Warning! Current implementation does not set any security manager.

Parameters
dir The directory for the plugins
clazz The class object of the interface
Throws
Exception Could throw an exception with overriden loadPluginWrapper.

Protected Methods

protected final void loadPlugin (File file, Class<T> clazz)

Loads single plugin. Performs actual checking for the Plugin-Class, locating the plugin class and examining its interfaces on behalf of loadPlugins.

The code will throw PluginJarException if the entry is missing, ClassNotFoundException if the class cannot be located and InterfaceNotFound if the class does not have the plugin interface as it's direct ancestor.

Parameters
file The .jar file with the plugin
clazz The class object of the interface
Throws
Exception The problem reported by the class matching algorithm

protected void loadPluginWrapper (File file, Class<T> clazz)

Wrapper for loadPlugin. Catches most of the exceptions loadPlugin might have thrown. This way problem with one faulty plugin will not prevent other plugins to load. Implementers of the Plugin Engine might decide to handle this in other way, they should, however, call the loadPlugin with the arguments provided, or no plugins will be loaded.

Parameters
file The .jar file with the plugin
clazz The class object of the interface
Throws
Exception In overriden methods: the problem reported by the class matching algorithm.
